# Lamo — Архитектура чат-приложения с локальным LLM

## Обзор

Создаём масштабируемое SwiftUI приложение для чата с локальной LLM-моделью. Архитектура построена на протоколах, Dependency Injection и чистом разделении ответственности — легко добавлять новые провайдеры, фичи и экраны.

## Стек

- **SwiftUI** + **SwiftData** для persistence
- **Apple Intelligence / Core ML** для локальной LLM
- **AsyncStream / AsyncSequence** для streaming
- **MarkdownUI** (или встроенный Markdown) для рендеринга ответов
- **Modern Swift concurrency** (async/await, actors)

## Детали по слоям

### 1. Models (SwiftData)

**Message.swift**
```swift
import SwiftData

@Model
final class Message {
    var id: UUID
    var content: String
    var role: MessageRole       // .user | .assistant
    var timestamp: Date
    var isStreaming: Bool       // true пока ответ не завершён
    
    @Relationship(inverse: \Conversation.messages)
    var conversation: Conversation?
}

enum MessageRole: String, Codable {
    case user, assistant
}
```

**Conversation.swift**
```swift
@Model
final class Conversation {
    var id: UUID
    var title: String
    var createdAt: Date
    var updatedAt: Date
    var messages: [Message]
}
```

### 2. Services

**LLMProvider.swift** — Протокол для любого LLM-провайдера
```swift
protocol LLMProvider {
    var name: String { get }
    func streamResponse(
        messages: [ChatMessage]
    ) -> AsyncStream<StreamingToken>
}
```
Любой новый провайдер (OpenAI, Anthropic, локальная модель) — просто новая структура, conforming к `LLMProvider`.

**StreamingToken.swift**
```swift
enum StreamingToken {
    case text(String)      // Часть текста
    case done              // Конец ответа
    case error(Error)      // Ошибка
}
```

**ChatService.swift** — Оркестратор
- Принимает `LLMProvider` через init (DI)
- Отправляет сообщения → получает streaming → сохраняет в SwiftData
- Управляет состоянием streaming (отмена, ошибки)

### 3. ViewModels

**ChatViewModel.swift** — @Observable
```swift
@Observable
final class ChatViewModel {
    var messages: [Message] = []
    var inputText: String = ""
    var isStreaming: Bool = false
    
    private let chatService: ChatService
    private let conversation: Conversation
    
    func send() async { ... }       // Отправить и начать streaming
    func stopGeneration() { ... }   // Остановить streaming
}
```

**ConversationListViewModel.swift** — @Observable
```swift
@Observable
final class ConversationListViewModel {
    func createConversation() -> Conversation { ... }
    func deleteConversation(_ conversation: Conversation) { ... }
}
```

### 4. Views

- **ChatView**: `ScrollViewReader` + LazyVStack сообщений + поле ввода с send-кнопкой. Автоскролл к новому сообщению.
- **MessageBubble**: Разные стили для user/assistant. Ассистент — слева, пользователь — справа.
- **MarkdownRenderer**: Используем `AttributedString` или第三方 `MarkdownUI` для рендеринга кода, списков, жирного и т.д.
- **TypingIndicator**: Анимированные точки пока LLM "думает" до первого токена.
- **ConversationListView**: `NavigationStack` → `ChatView`. Swipe для удаления, long press для переименования.

### 5. Streaming

```
LLMProvider.streamResponse() → AsyncStream<StreamingToken>
    ↓
ChatService积累了 токены в строку
    ↓
Обновляет Message.content в SwiftData в реальном времени
    ↓
SwiftUI реагирует через @Observable
```

### 6. Design System

**Theme.swift** — единый стиль:
- Цвета: background, bubbleUser, bubbleAssistant, accent
- Шрифты: body, code, header
- Spacing: constants для отступов

## Верификация

1. Проект компилируется без ошибок в Xcode 26.2
2. Запускается на симуляторе — виден NavigationStack с пустым списком диалогов
3. Можно создать новый диалог
4. Можно отправить сообщение (пока с заглушкой от LLM)
5. Streaming работает — текст появляется по токенам
6. Markdown рендерится корректно (код, списки)
7. История сохраняется между перезапусками (SwiftData)
